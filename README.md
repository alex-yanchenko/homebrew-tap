# homebrew-tap

A personal [Homebrew](https://brew.sh) tap for my command-line tools, so they install and
update the same way on any machine:

```bash
brew install alex-yanchenko/tap/<tool>   # the tap is added automatically on first use
brew upgrade                             # updates every installed tool
```

## Tools

| Tool | What it is | Status |
|---|---|---|
| **ccsearch** | Search your Claude Code sessions by content | ⏳ Live once ccsearch `v0.1.0` is released (see below) |
| **skaldr** | Render a YAML content file into a self-contained HTML report | ⏳ Live once skaldr is published to PyPI |

**kvasir is deliberately not here.** It's a multi-part system (channel binary + Chrome
extension + `/kvasir` skill) whose parts must stay on the same version; installing only the
channel binary via brew would let `brew upgrade` bump one part out of lockstep. kvasir
updates itself instead — run `kvasir upgrade`.

> For the Python tools, `uvx <tool>` or `pipx install <tool>` (once each is on PyPI) is an
> equivalent install path. The tap additionally pulls in the external binaries a tool needs
> (e.g. ripgrep + fzf for ccsearch) and fits the `brew upgrade` habit.

## How updates flow

1. Each tool uses **release-please**: conventional commits accumulate into a release PR;
   merging it (and publishing the draft) cuts a versioned release — no manual building.
2. **`update-formulae.yml`** in this tap runs daily (and on demand), reads each tool's
   latest published release, and rewrites the formula's `url` + `sha256` from the release
   asset's API digest. It commits only to this tap, so no cross-repo token is involved.
3. `brew upgrade` on any machine then picks up the new version.

## Activation checklist

The formulae are scaffolded but go live only when their sources exist:

- **ccsearch** — merge [alex-yanchenko/ccsearch#2](https://github.com/alex-yanchenko/ccsearch/pull/2),
  merge the release-please PR, and publish the `v0.1.0` draft release. On the next run,
  `update-formulae.yml` fills `Formula/ccsearch.rb`'s real `sha256` (it ships as a placeholder).
- **skaldr** — finish PyPI trusted publishing and cut a release. Then add `Formula/skaldr.rb`
  (installed from PyPI, with `resource` blocks generated by `brew update-python-resources`)
  and a PyPI-sourced branch to `update-formulae.yml`.
- Confirm this repo is named **`homebrew-tap`** and is public, so `brew tap alex-yanchenko/tap`
  resolves.

## Layout

```
Formula/            # one <tool>.rb per formula
.github/workflows/
  update-formulae.yml   # daily: bump formulae to latest releases
  tests.yml             # brew style (rubocop) on push/PR
```
